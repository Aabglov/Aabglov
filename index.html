This a demo I've been working on to improve GPS tracking on mobile devices. The dots above move about the area randomly while a neural network learns how to predict their next positions from their speed and direction. The squares represent where the network believes the dots to be based on the last location, direction and speed. To represent more realistic data there's also 20% random noise in both direction and speed.

I think this same technology could be applied to mobile sensors such as the accelerometer to reduce the reliance of location-tracking applications on GPS. This would result in improved battery life while using the apps and the ability to use apps in areas of weak GPS signal.

Current Projects

I'm working with a local government agency to improve the way they collect data on trees. I'm creating a Recurrent Neural Network that will learn to predict the height of trees based on the heights of other trees in the immediate area.

I've played around with a lot of different Machine Learning libraries, but by far my favorite is Theano for Python. I love the control, the power and the flexibility. However that same flexibility and power can lead to code that's difficult to understand and debug. I've seen add-on modules like Lasagne which help standardize things, but at the cost of control. So I've started my own which you can check out here: https://github.com/Aabglov/TheanoSeq2Seq.

It began as just a sequence-to-sequence program, but has balooned into my personal Machine Learning library. I'm currently working on automating the initialization of multiple layers and debugging an LSTM that's learning to rap.
